<objective>
Fix the blocking `.gitignore` defect that makes `skills/north-star/SKILL.md` and `skills/goals/SKILL.md` invisible to git, and close the `profile.md` safety-net gap. This plan touches only the safety infrastructure — it does NOT commit the skill files themselves (that is plan 02-02, which depends on this plan landing first).

The root cause: all six vault-content directory patterns in `.gitignore` are unanchored (`north-star/`, `goals/`, `daily/`, `projects/`, `council/`, `archive/`). Unanchored gitignore patterns match at any directory depth, so `north-star/` also matches `skills/north-star/` and `examples/north-star/`. Anchoring each pattern with a leading slash (`/north-star/`) restricts it to the repo root, which is the only place vault content would ever land in the engine.

Purpose: Without this fix, ONBD-04 (`/kairos-north-star`) and ONBD-05 (`/kairos-goals`) cannot ship — the skill files physically cannot enter the engine repo. This plan unblocks plan 02-02. It also extends the M002 gate so that "tests pass" actually means "the skills are committed," not merely "the files exist on this one machine."
Output: An anchored `.gitignore`, `profile.md` registered in both `.gitignore` and `hooks/content-globs.txt`, and two new git-tracking assertions in `test/m002.test.sh` (RED until 02-02 commits the skills).
</objective>
