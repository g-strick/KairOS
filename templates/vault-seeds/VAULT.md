# Kairos Vault

Your notes live here. Engine tooling is under `_kair/` (PKM meta convention — underscore sorts away from daily folders).

## Your content

- `inbox/` — unprocessed captures (`/kair-capture`)
- `daily/` — today's note (`/kair-daily`)
- `projects/` — active projects (one `.md` per multi-step outcome)
- `reference/` — reference material
- `next-actions.md`, `waiting-for.md`, `someday-maybe.md` — GTD lists (append only; process skills never create new list files)
- `profile.md` — from `/kair-onboard`

## Navigation

- `INDEX.md` — agent L0 map of the vault (update when adding top-level folders)
- `VAULT.md` — this file (human orientation)

## Engine (`_kair/`)

Synced from KairOS via `/kair-update`:

- `_kair/skills/`, `_kair/styles/`, `_kair/hooks/`, `_kair/templates/`, `_kair/agents/`
- `.cursor/skills/`, `.claude/skills/`, `.cline/skills/` — editor slash-command adapters (vault root; required by each tool)

`AGENTS.md` stays at the vault root — the AI reads it first.
